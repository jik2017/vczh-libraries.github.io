Getting started ...

Any GacUI project is highly recommended to create using XML resources. You can refer to the CppXml demo. Generally a GacUI project can be created by the following steps:

  • Create a VC++ project using Visual Studio 2015 or higher version, and add all source codes from the Release folder to your project.
  • Prepare your XML resource.
  • Generate necessary C++ classes and the binary resource file from your XML resource using GacGen.exe. This command line tool is required to run when you modify your XML resource.
  • Complete the WinMain and GuiMain functions, load the binary resource file, create the main window and run.

Notes : The binary resource file don't necessarly be a file. You can embed the file as a resource linking to your exe. After you load it in the memory, you can use MemoryStream instead of FileStream to load the resource.

Different ways to create a GacUI project

Category Id Demo Target
CLI C1 To create a cross-platform CLI tool working with GacUI-compatible types
C2 To create a cross-platform CLI tool to deal with Workflow script
C3 To create a cross-platform CLI tool to deal with GacUI XML resource
GUI G1 Cpp To create a C++ GacUI project purely by C++
G2 To create a C++ GacUI project by GacUI XML resource, and GacGen.exe will compile UI to Workflow byte code. You need the metadata to load the precompiled binary resource, and link to GacGen-generated C++ code.
G3 Xml To create a C++ GacUI project by GacUI XML resource, and you want your application has the ability to load the precompiled binary resource at runtime, without having to link any GacGen-generated C++ code.
G4 CppXml To create a C++ GacUI project by GacUI XML resource, but you want GacGen.exe to translate everything about UI in the XML to C++ code, so that you don't need metadata to load the precompiled binary resource. It reduces about 10-20 MB in the binary, and significantly improves the performance.
